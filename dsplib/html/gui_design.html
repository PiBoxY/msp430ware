Design

Filter design in DSPLib GUI is implemented using the Filter Structure and Design Specifications panels on the left-hand side side of the GUI. There are five primary components of the filter structure design:

  • The Q-Type, specifying the amount of precision that the exported coefficients will retain; the available options consist of Q15 or Q31.
  • The structure, which sets the available options as those of a finite impulse response (FIR) or infinite impulse response (IIR) filter type.
  • The filter type, which sets the target bandwidth.
  • The method, which allows the selection of several different windowing patterns for FIR filters or design methods for IIR filters.
  • The name of the filter, which will set the generated code variable names as well as the file name of generated .c and .h files if saved.

Supported Methods

The design panel has options specific to the type of filter. For both FIR and IIR functions, four types of filter are supported: bandpass, lowpass, highpass, and stopband. For FIR filters, Boxcar, Triangle, Blackman, Hamming, Hann, Bartlett, Flat-top, Parzen, Bohman, Blackman-Harris, Nuttall, Barthann, Kaiser, Gaussian, General Gaussian, Exponential, and Tukey methods are supported. For IIR filters, Buttersworth, Chebyshev, Chebyshev II, Elliptical, and Bessel methods are supported.

Unit Types

For the unit of the filter design, two types are supported:

  • Normalized, where the frequency range spans [0, 1] where 1 symbolizes is the Nyquist Frequency and is expressed in cycles/sample. For example, a filter expressed in Hz that has a sampling rate of 10,000 Hz with a passband ranging from 2,000 Hz to 4,000 Hz would have normalized critical frequencies .4 and .8.
  • Hertz, where the frequency range spans [0, Fs / 2]. For this parameter mode, a sampling frequency must be given as well.

Band Types

Four types of target frequency are supported:

  • Bandpass, where two critical frequencies must be given with a passband created between the two. Bandpass filters must not have critical frequencies at either 0 or the Nyquist Frequency.
  • Bandstop, where two critical frequencies must be given with a stopband created between the two. Bandstop filters must not have critical frequencies at either 0 or the Nyquist Frequency. Bandstop filters must be even-ordered because odd-ordered filters cannot fulfill the requirement of zero amplitude at the Nyquist frequency; odd-ordered filters input into DSPLib GUI will have one added to the order to make it even.
  • Lowpass, where one critical frequency must be given with a passband region created from [0, FCritical].
  • Highpass, where one critical frequency must be given with a passband region created from [FCritical, FNyquist]. Highpass filters must be even-ordered because odd-ordered filters cannot fulfill the requirement of zero amplitude at the Nyquist frequency; odd-ordered highpass filters input into DSPLib GUI will have one added to the order to make it even.

Filter Order

Filter order for DSPLibGUI can range between [1, 500] for FIR filters and [1, 25] for IIR filters. Higher order filters generally achieve a better attenuation pattern, but at a higher runtime cost; FIR filters will linearly increase in runtime by the order. IIR filters typically require lower runtime, but have increased sensitivity due to feedback.
