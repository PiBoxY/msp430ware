XT2 in BYPASS Mode

Associated constant in the API stacks: USB_XT2_BYPASS_MODE. (Not reported in the USB descriptors.)

Suggested default: leave it unchecked, and allow the on-board XT2 crystal to provide the signal source.

Some devices support XT2 bypass operation with external clock inputs that reside on a different external supply domain. The API enables the XT2 oscillator to be placed in Bypass mode when external clock signal is preferred.

If this box is checked, the API will set the XT2 crystal in BYPASS mode and external signals can be passed in via the XT2IN pin. If the box is not checked and the selected device has the second crystal oscillator, XT2, it will be used as the signal source.
